react에서 IE9/11 지원하기

2020-01-20

서론


우리나라에서는 특히 웹 사이트 같은 경우에 여전히 IE를 사용하는 사용자들이 많기 때문에 웹 서비스를 개발하는 경우 IE를 고려하지 않을 수가 없습니다. 

react로 웹 프로젝트를 진행하는 경우 기본적으로 IE를 지원하지 않기 때문에 별도의 추가 설정이 필요합니다. 

오늘은 react에서 IE9/11을 지원하는 방법에 대해서 알아보겠습니다. 


연구내용


-패키지 매니저는 npm을 사용하고 있습니다. 

npx create-react-app 명령어를 통해 테스트를 진행할 리액트 프로젝트를 생성해줍니다. 

npx create-react-app my-app --template typescript


react-app-polyfill 패키지를 설치해줍니다.  react-app-polyfill에서는 다양한 브라우저를 지원하기 위한 여러가지 도구들이 포함되어 있습니다. 

npm install react-app-polyfill


my-app 프로젝트/src/index.tsx 파일을 수정하기 위해 열어줍니다.  파일 상단에 polyfill 설정을 추가해줍니다. 

만약 IE11만 지원해도 된다면 react-app-polyfill / ie11 으로 작성해주면 됩니다. 그러나 우리는 IE9도 지원해야 하기 때문에 ie9라고 적어주었습니다. 

주의해야 할 점은 반드시 파일의 첫줄에 적어줘야 합니다. 

import 'react-app-polyfill/ie9' //ie9~
import 'react-app-polyfill/stable'; // 대상 브라우저에 필요한 폴리필만 포함


브라우저 목록을 추가해주기 위해 my-app/package.json 파일을 열어줍니다. 하단의 browserslist의 development를 다음과 같이 수정해줍니다. 

"development": [
      "ie 9",
      "ie 10",
      "ie 11",
      "last 1 chrome version",
      "last 1 firefox version",
      "last 1 safari version"
    ]


예외)  my-app/public/index.html에 추가

(이 부분은 SCRIPT438: Object doesn't support property or method 'isArray' react-with-addons.js, line 4 character 31 와 같은 스크립트 에러 발생시 추가해도 됩니다.  테스트 결과 없어도 정상적으로 동작하는 것을 확인하였습니다.) 

<meta http-equiv="X-UA-Compatible" content="IE=edge">


그러나 IE11에서  SCRIPT5022:SyntaxError  가 자꾸 발생하는 문제가 생겼습니다.  검색 결과 react-scripts 버전과 연관이 있다고 해서 react-scripts@3.3 대신 3.2를 설치해주었습니다. 

npm uninstall react-scripts
npm install react-scripts@3.2


그러나 IE11에서는 정상적으로 작동하지만 IE10, 9에서 Map, set 과 관련된 에러가 발생하였습니다. 

이 문제는 react 공식홈에 잘 설명되어 있었습니다. 

React 16 depends on the collection typesMapandSet. If you support older browsers and devices which may not yet provide these natively (e.g. IE < 11) or which have non-compliant implementations (e.g. IE 11), consider including a global polyfill in your bundled application, such ascore-jsorbabel-polyfill.


그전에 core-js, raf 패키지를 설치해줍니다. 

npm install core-js
npm install raf

my-app/src/index.tsx 파일을 수정하기 위해 열어, 아래를 추가해줍니다.  필요한 부분을 import 해줍니다. 

import 'core-js/es/map';
import 'core-js/es/set';
import 'raf/polyfill';

그래도 IE9 기반에서 여전히 (SCRIPT5009: 'Map' is undefined) 에러가 발생하였습니다. 

검색 결과 현재 dev 서버가 IE9/11을 지원하지 않는다고 합니다. 즉 npm start을 실행하게 되면 dev서버가 실행되기 때문에 에러가 발생했던 것입니다. 

https://github.com/facebook/create-react-app/issues/5674 참고

"scripts": {
   "start": "react-scripts start",
   "build": "react-scripts build",
   "test": "react-scripts test",
   "eject": "react-scripts eject"
 },

build를 통해 생성된 파일을 실행시켜 보겠습니다. 

npm run build

빌드가 성공적으로 진행되면 myapp 루트 디렉토리 하단에 build 폴더가 생성되게 됩니다. 

이 생성된 파일을 확인할 수도 있지만 serve설치후  serve -s build을 통해 정적 사이트로 확인할 수 있습니다. 

$ serve -s build

결론

 react에서 IE9/11을 지원하는 방법에 대해서 알아보았습니다.

IE9/11 지원에 대해 검색 했을때, 단순히 react-app-polyfill 패키지만 사용하면 된다고 설명이 주로 나와있었기 때문에, 쉬울꺼라고 생각했었습니다.  

그러나 내용이 부족한 참고자료도 있었지만, 버전에 따라 변수가 많이 생기기 때문에 에러가 많이 발생했었습니다. 

위에서 기술되어 있지 않는 에러가 추가적으로 발생한다면 버전 문제가 아닐까 생각해봐도 될꺼같습니다.
